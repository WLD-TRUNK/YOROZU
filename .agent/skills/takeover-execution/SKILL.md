# 引き継ぎ実行

目標:
- 直近指示を固定し、再解釈のぶれを止める。
- 結論の押し付けを避け、比較可能な試算を返す。

手順:
1) 直近指示を3行で再定義する。
2) `固定条件` と `可変条件` を分離する。
3) 固定条件は編集禁止として明示する。
4) 不足情報を列挙し、欠損は `仮定` としてラベル付けする。
5) シナリオを2〜4本に限定し、同一期間で試算する。
6) 結果を `前提表` `試算表` `差分要因` `未確定項目` の順で提示する。
7) 次アクションを番号付きで1〜3件提示する。

必須ルール:
1) 「ここだけ変える」指示がある場合、対象外は編集しない。
2) 感情的な文面でも煽り口調を引き継がない。
3) 金額、日付、プラン名には出典または前提ラベルを付ける。
4) 未確定値を確定値として扱わない。

出力テンプレート:
1) 要件固定:
- 固定条件:
- 可変条件:
2) 試算前提:
- 前提A:
- 前提B:
3) シナリオ比較:
- S1:
- S2:
4) 未確定:
- 確認が必要:

トリガー語彙（例）:
- 引き継ぎ
- 指示を守らない
- 要件逸脱
- ここだけ変える
- 整理して試算

- `.agent/rules/STRUCTURE_POLICY.md`
- 必要時のみ `一時出力フォルダ/` に作業メモを出力する
