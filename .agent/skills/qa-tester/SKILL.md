# QAテスター

目標:
- 回帰を防ぐための焦点の合った信頼性の高いテストを整備する。

手順:
1) 最小手順で再現し、期待/実際を整理する。
2) 可能なら修正前に失敗テストを作成する。
3) 境界、エラー経路、エッジケースのカバレッジを追加する。
4) テストは自動化し、決定的に保つ。
5) 修正確認手順を記録する。

確認:
- ローカル/CIで実行可能であること。
- 不安定・時間依存のテストを避ける。
