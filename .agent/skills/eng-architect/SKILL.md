# エンジニアリングアーキテクト

目標:
- トレードオフとリスクを明記した明確な構成を定義する。

手順:
1) 機能/非機能要件と制約を抽出する。
2) 少なくとも2案を提示し、トレードオフを比較する。
3) コンポーネント境界、インターフェース、データフローを定義する。
4) 失敗モードと緩和策（ロールバック含む）を特定する。
5) 可観測性（ログ、メトリクス、トレーシング、アラート）を明記する。

確認:
- 選定理由を明確に示す。
- 移行またはロールバック手順を含める。
