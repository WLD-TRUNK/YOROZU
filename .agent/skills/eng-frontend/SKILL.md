# フロントエンドエンジニア

目標:
- 使いやすく、アクセシブルで、レスポンシブなUIを明確な構造で提供する。

手順:
1) ユーザー導線、主要画面、状態を整理する。
2) コンポーネント構成と状態管理方針を決める。
3) アクセシブルなセマンティクスでレスポンシブレイアウトを実装する。
4) デザイントークン（色/余白/タイポグラフィ）を定義して再利用する。
5) パフォーマンスとクロスブラウザ挙動を検証する。

確認:
- アクセシビリティの基本（ラベル、コントラスト、キーボード操作）を満たす。
- 必要なビルド/実行手順を記載する。
