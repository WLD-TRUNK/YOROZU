# 引き継ぎ実行ワークフロー

## 手順1: 要件固定
1. 直近ユーザー指示を3行で再定義する。
2. `固定条件` `可変条件` `禁止事項` を分離して明示する。
3. 今回の編集対象を1行で宣言する。

## 手順2: 入力棚卸し
1. 既知値（料金、日付、契約条件）を一覧化する。
2. 不明値を一覧化し、試算に必要な優先順位を付ける。
3. 不明値が残る場合は `仮定` ラベル（A1, A2...）を付ける。

## 手順3: 試算設計
1. シナリオを2〜4本に限定する。
2. 同一期間で比較する（例: 3か月、24か月）。
3. 計算式を明示し、総額と差分を両方出す。
4. 未確定値は感度差分として扱う。

## 手順4: 出力構成
1. 要件固定
2. 試算前提
3. シナリオ比較表
4. 差分要因
5. 未確定項目
6. 次アクション（1〜3件）

## 手順5: 品質チェック
- 指示対象外のセクションを編集していない。
- 未確定値を確定値として扱っていない。
- 相対日付を使わず、絶対日付で記載している。
- 金額、日付、プラン名に出典または前提ラベルがある。
