# YOROZU リポジトリ作成ルール

このルールは、YOROZUワークスペース内で新しいプロジェクトやリポジトリを作成する際に**常に**適用されます。

## 1. 作成場所
- **原則**: すべての新しいリポジトリは `YOROZU_PROJECT/` ディレクトリの直下に作成する。
- **構造**: `YOROZU_PROJECT/{project-name}/` が基本単位となる。
- **パッケージ思想**: 各プロジェクトは「自己完結したパッケージ」として扱う。必要な設定や依存関係はそのディレクトリ内で完結させること（グローバルな設定はルートの `.agent` が担うが、個別の実行環境は混ぜない）。

## 2. 命名規則
- **分析**: プロジェクトの内容（コード、README等）を分析し、**意味のある名前**をつける。
- **形式**: **UPPER_SNAKE_CASE** を厳守（例: `DAY_REPORT`, `NOTION_KNOWLEDGE_VAULT`）。
- **記号**: 区切りは半角アンダーバー `_` のみ許可。ハイフンや空白は使用しない。
- **日本語表記**: ディレクトリ名は英語大文字に統一し、日本語名は README で補足する。
- **実名禁止**: 個人名や実名は使わず、匿名化した名称に置き換える。
- **生成物の統一**: 新規に作成する成果物のディレクトリ名も **UPPER_SNAKE_CASE** を適用する。
- **作成物（ファイル/スクリプト）**: ユーザー/AI が作成するファイル名は `UPPER_SNAKE_CASE`。自動生成・インストールは小文字のまま扱う（詳細は `.agent/rules/STRUCTURE_POLICY.md`）。
- **リネーム**: 既存フォルダ名が規則に合わない場合、準拠名への変更を提案する。

## 3. Git構成
- **独立性**: 各プロジェクトは独立した `.git` リポジトリとして初期化する（モノレポではなく、マルチレポ管理）。
- **除外設定**: ルートの `.gitignore` とは別に、プロジェクト内に専用の `.gitignore` を配置する。

## 4. 集約サーバー（管理プロジェクト）
- **目的**: 複数の独立リポジトリを AI で横断管理するための集約サーバーを用意する。
- **配置**: `YOROZU_PROJECT/CONTROL_HUB/` を単独リポジトリとして管理する。
- **位置づけ**: 管理用途であり、他プロジェクトの実装や依存を直接抱え込まない。

## 5. 自動化推奨
- **ワークフロー優先**: 手動手順ではなく、可能な限り既存のワークフロー（例: `/create-yorozu-project-complete`）を使用する。これによって「YOROZU化（ブランド適用、初期設定）」を自動的に完了させる。

## 6. GitHub連携
- **同期**: `gh repo create` を使用してGitHub上にリポジトリを作成し、初期プッシュまで行う。
