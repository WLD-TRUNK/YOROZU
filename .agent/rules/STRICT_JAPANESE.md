# 完全日本語出力ルール

ユーザーとのチャットだけでなく、**作成するすべての成果物（ドキュメント、コミットログ、コード内コメント等）において日本語を使用すること**を強制します。

### 具体的な適用範囲

1. **ドキュメント全般**
   - README.md
   - walkthrough.md
   - implementation_plan.md
   - その他、ユーザー向けの説明ファイル
2. **コミットメッセージは日本語で記述する。**
3. **コメントおよびドックストリング（docstring）は日本語で記述する。**
4. **テンプレートから生成される提出物**
   - リリースノート
   - ヘッダー生成プロンプト
   - 図解用テキスト（図中ラベルを含む）
   - など、提出物に転記される文面は日本語で統一する。

### 英語の限定許容

- **許容対象**: コマンド、ショートカット、フック名、識別子、ファイル名、パス、API名、エラーメッセージ、固有名詞。
- **英文の扱い**: 英語の**文**を含める場合は、直後に日本語訳を併記する（英語のみは不可）。
- **英語主体の資料**: 英語が必須のときは「英語本文 + 日本語訳」をセットで記載する。

### 文字コード（エンコーディング）

- **必須**: 成果物のテキストは **UTF-8（BOMなし）** を使用する。
- **例外**: 既存ファイルがUTF-8以外の場合は、勝手に変換せず事前に確認する。

### 検索結果（Web等）の取り扱い

- 検索由来の英語コンテンツを**そのまま貼り付けない**。
- 原則、検索結果は**日本語で要約**して提示する。
- 仕様・役職・価格・規制・互換性など、変化し得る事実は **[as-of: YYYY-MM-DD]** を付与する（ユーザーのタイムゾーン基準）。
- 参照元は（可能な限り）引用情報として追跡可能にする。


※ ただし、上記以外の本文・見出しは日本語に統一する。

## 運用チェック（毎回の自己点検）

- [ ] 見出し・本文は日本語で統一されている（英語見出しが残っていない）
- [ ] 英語文がある場合、日本語訳が直後に併記されている
- [ ] 検索結果の文章を英語のまま貼っていない（必要最小限の固有名詞・識別子のみ）
- [ ] 変化し得る事実に [as-of: YYYY-MM-DD] を付けた
- [ ] 提出物に転記される文面がすべて日本語になっている

## 例外（英語出力の明示指示）

ユーザーが**明示的に英語での出力を指示した場合のみ**、英語出力を許可する。
